Overview

The FizzBuzz exercise is one of the straightforward and most well-known coding challenges out there. The rules are simple; Write a short program that prints each number from 1 to 100 on a new line. For each multiple of 3, print "Fizz" instead of the number. For each multiple of 5, print "Buzz" instead of the number. For numbers which are multiples of both 3 and 5, print "FizzBuzz" instead of the number. The secondary objective of the challenge is use as few characters as possible in your code.
Writing a program which prints the correct statement for the numbers 1 to 100 was easy to achieve using a ‘for loop’ , a series of ‘if’ statements and the modular operator. To me, the challenge lay in accomplishing this with as few characters as I could. I took the approach of ‘every little helps’ and implement whatever required the fewest characters.
In my for loop, the range was set as “i < 101” simply because it is 1 less character than “i <= 100” and the statements are equivalent. In all of my ‘if’ statements I used the equality operator instead of the strict equality operator because it made no difference in these circumstances and saved a character with each use. To check whether a number was a multiple of both 3 and 5, I checked whether it was a multiple of 15. Only checking if it is a multiple of 1 number takes fewer characters and as it is the lowest common numerator, any number which is a multiple of 3 and 5 must also be a multiple of 15.
In my second attempt I used the ‘console.log()’ method instead of ‘innerHTML’. This meant the result would not be printed to the web page and could only be viewed in the console however it saved a significant number of characters. Instead of ‘if’ statements I used what is known as the ternary operator. This performs the same function but is much more efficient.
In my third attempt the only alteration I made was assigning the words “Fizz” and “Buzz” as variables. This only saved a single character, but every little helps.
